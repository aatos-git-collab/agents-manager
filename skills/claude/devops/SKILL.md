# WebBuilder Infra Guide

## Compose file structure
```
docker-compose.yml          # base: postgres, redis, traefik, api, frontend, worker
docker-compose.prod.yml     # overrides: dind sidecar, replicas, TLS certs, monitoring
docker-compose.test.yml     # overrides: test database, no dind needed for unit tests
```

## Docker-in-Docker (dind) — Production only
dind container runs `docker:dind` with TLS. API and workers connect via:
```
DOCKER_HOST=tcp://dind:2376
DOCKER_TLS_CERTDIR=/certs
```
TLS certs are auto-generated by dind and mounted to the API/worker containers.

## NEVER do this
- `docker ps` or `docker compose` on the HOST — everything runs in containers
- Install dependencies locally — Docker builds handle everything
- Use `container_name` on any service with `deploy.replicas` (conflict)

## Key production configs
- Worker: `deploy.replicas: 2` — scaled independently from API
- builder-init: `restart: "no"` — runs once on deployment, exits
- Volumes: bind-mount local dirs (postgres-data, redis-data) — backed by EBS/network storage in prod
- Traefik: dynamic config via file provider ( `./traefik/dynamic.yml`)

## Traefik dynamic config
When a project container starts, its preview URL is added to `dynamic.yml`:
```yaml
http:
  services:
    project-{id}:
      - loadBalancer:
          servers:
            - url: "http://{container-ip}:{port}"
  routers:
    project-{slug}.{domain}:
      - rule: "Host(`{slug}.builder.io`)"
        service: project-{id}
```
Then `traefik-compose reload` or SIGHUP to hot-reload.

## Healthcheck requirements
Every service MUST have a healthcheck so `depends_on: condition: service_healthy` works.
- postgres: `pg_isready`
- redis: `redis-cli ping`
- traefik: `wget -q --spider http://localhost:8080/api/version`
- api/worker: `curl -f http://localhost:{port}/api/health/live`

## Build & test in Docker
```bash
docker compose -f infra/docker-compose.yml -f infra/docker-compose.test.yml up --build test
```
