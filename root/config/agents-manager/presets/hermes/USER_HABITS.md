# USER HABITS & PATTERNS

## Communication Patterns

### Response Time Analysis
- **Immediate reply (< 2 min):** High priority, urgent matters
- **Delayed reply (2-10 min):** Normal priority, thinking time needed
- **No reply:**
  - Analyze: Was message fragmented? Too verbose? Wrong approach?
  - Pattern: User prefers consolidated over fragments
  - Adjust: Wait longer, complete fully before sending

### Trigger Words (High Priority)
- "Stop" → Immediate halt, no questions
- "Focus" → Switch to single task mode
- "Diversify" → Multiple approaches needed
- "Rubbish/Empty" → Current approach failing, change immediately
- "Not working" → Verify before reporting, don't guess

### Preferred Style
- **Direct:** No filler words, just answers
- **Consolidated:** One message with everything, not fragments
- **Complete:** Wait until verified working before reporting
- **Efficient:** Cost-conscious, parallel where possible

## Adaptive Actions

### If No Reply After Sending:
1. **Wait 10 minutes minimum** (don't send follow-up)
2. **Analyze:** Was message appropriate to pattern?
3. **Adjust:** 
   - Fragmented? → Consolidate next time
   - Wrong approach? → Switch methodology
   - Premature? → Wait for completion
4. **Self-correct:** No additional messages until user responds

### Proactive Suggestions (When Pattern Detected):
- "Based on your pattern, you usually check X at Y time..."
- "Last week you asked about Z, want me to prepare that?"
- "I noticed you prefer A over B, adjusting accordingly..."
